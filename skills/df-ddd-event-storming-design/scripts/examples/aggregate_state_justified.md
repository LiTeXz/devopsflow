# 问题域边界
货架占用登记。

# 主体与协作场景
- 仓库管理员 -> 登记货架 -> 货架已登记 -> 货架选项读模型
- 仓库管理员 -> 暂停货架使用 -> 货架已暂停 -> 货架选项读模型移除该货架

# 领域事件
- 货架已登记
- 货架已暂停

# 命令
- 仓库管理员发起登记货架
- 仓库管理员发起暂停货架使用

# Policy
- 无

# 聚合设计

## 货架聚合

### 聚合属性

| 属性 | 业务方法使用 | 规则或事件依据 | 判断 |
| --- | --- | --- | --- |
| 货架身份 | 暂停货架使用读取它来定位聚合 | 聚合身份 | 保留 |
| 仓库身份 | 登记货架读取它来保护同仓库货架编号唯一 | 唯一性边界 | 保留 |
| 货架编号 | 登记货架读取它来保护同仓库唯一 | 同仓库编号不能重复 | 保留 |
| 当前状态 | 暂停货架使用读取并改变它 | 只有可用货架才能被暂停，货架已暂停事件由状态变化产生 | 保留 |

### 规则

- 登记货架时，业务方法检查同仓库编号不能重复。
- 暂停货架使用时，业务方法检查当前状态必须是可用，并产生货架已暂停。

# 读模型
- 货架选项由货架已登记、货架已暂停投影。

# 完整性检查
- 每个聚合属性都能被业务方法、唯一性规则或事件产生路径解释。
